## 🎯 ¿Qué es?
Una comparativa visual de cómo estructurar peticiones para Claude Code para evitar fallos de contexto y lógica.

## 🚀 Puntos Clave
- **El Método Arquitecto (Correcto)**: Definir el rol, analizar el negocio, diseñar la arquitectura y generar documentos base (PRD/Tech Spec) **antes** de tirar código.
- **El Error del SaaS (Incorrecto)**: Pedir todo de golpe ("hazme un SaaS con todo") genera errores de "Undefined architecture" y "Project scope missing".
- **ROI**: **ALTO.** Reduce el retrabajo y el consumo de tokens de razonamiento al dar instrucciones granulares.
