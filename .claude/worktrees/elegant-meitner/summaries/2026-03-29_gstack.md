# Novedad: gstack (virtual engineering team)

**Fecha**: 2026-03-30
**Categoría**: Ingeniería / Agentes IA
**Fuente**: Note `20260329-2118_nota.txt`

## 🎯 ¿Qué es?
Un proyecto Open Source liberado por Garry Tan (CEO de Y Combinator) que permite usar Claude como un equipo virtual de hasta 20 ingenieros coordinados. Licencia MIT.

## 🚀 Puntos Clave
- **Estructura jerárquica**: CEO, Eng Manager, QA, Security, Release Engineer.
- **Flujo determinista**: Think, Plan, Build, Review, Test, Ship, Reflect.
- **Productividad**: Capaz de generar entre 10k y 20k líneas de código diarias.
- **Éxito**: 49,000 estrellas en GitHub en dos semanas.

## 💼 Oportunidad Workflow Waves
- **Problema real**: Startups o proyectos que necesitan ingenieros especializados pero no tienen presupuesto para 10 personas.
- **Pitch**: "Te configuramos un equipo virtual de 20 agentes ingenieros que trabajan 24/7 en tu repositorio por una fracción del coste de un senior".

## ⚖️ ¿Vale la pena automatizar?
- **ROI**: **ALTO.** Si desarrollas software habitualmente (>5h/semanales), el tiempo que ahorras en boilerplate y QA se recupera en una semana. No es para tareas de 5 min aisladas, es para el flujo de construcción continuo.
