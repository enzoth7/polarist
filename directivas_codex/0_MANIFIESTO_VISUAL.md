# MANIFIESTO VISUAL Y ESTÉTICO DE POLARIST
> ⚠️ **REGLA ESTRICTA PARA AGENTES AI** ⚠️
> Este archivo es la "Constitución Visual" del proyecto basada en el Brand Kit Oficial (`Brand Kit/Polarist_BrandKit_B_Sutil.html`).
> **BAJO NINGUNA CIRCUNSTANCIA** debes usar colores, radios o fuentes fuera de estos parámetros.

## 1. La Paleta de Colores Oficial ("Bento & Apple Sutil")
La app debe transmitir autoridad de consultoría y energía de IA.
- **Background Primario:** `#010101` (Pure Black).
- **Texto / Fondo Secundario:** `#F6F6F6` (Off-white). Nunca blanco puro (`#FFFFFF`) a menos que sea un acento extremo.
- **ACENTO / CTA:** `#CAFE5B` (Polarist Green). Reservado para botones principales y datos clave.
- **Navegación / Glass:** `rgba(255, 255, 255, 0.02)` con `blur(24px)` y borde `rgba(255, 255, 255, 0.08)`.

## 2. Tipografía (El "Dúo Dinámico")
- **Titulares Editoriales (30%):** `Arno Pro` (Serif). Para títulos que requieren autoridad y elegancia.
- **Titulares de Impacto (70%):** `Sequel Sans Bold`. Para impacto técnico y directo.
- **Cuerpo de texto:** `Sequel Sans Regular` (Weight 400). **PROHIBIDO** usar pesos intermedios para párrafos.

## 3. Radios y Contenedores (Sistema Bento)
- **Radio Global / Botones / Inputs:** `4px` (Sutil).
- **Tarjetas (Cards):** `24px` (Bento style).
- **Pills:** `999px` para elementos fluidos.
- **Espaciado:** Sistema base 8px. Usar múltiplos exactos (8, 16, 24, 32, 48, 64...).

## 4. Estilo de Componentes
- **Botones:** Formato Pill o radio 4px según el contexto. Siempre con transición suave `scale(1.02)` on hover.
- **Cards:** Efecto glassmorphism sutil. Fondo casi transparente, borde mínimo, elevación por hover.

## 5. REGLA SUPREMA: ANTI-CONTENEDORES
- **Whitespace > Bordes.** Separar contenido por espacio en blanco y jerarquía tipográfica.
- Prohibido anidar cajas innecesarias. El diseño debe ser chato, espacioso y limpio como Apple.
- Si dudas en poner un borde, no lo pongas.
